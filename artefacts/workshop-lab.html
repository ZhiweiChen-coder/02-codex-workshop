Slide 3

Demo 1: Clarify The Idea

Start before code. Let Codex turn a rough idea into a product direction everyone can understand.

I want to build a simple recipe website for busy people who want to find meals based on ingredients they already have.
Help me clarify the product idea.

Look for: target user, problem, MVP, features, and first task.

Slide 4

Demo 2: Ask For A Plan

The key habit: do not ask Codex to rush into changes. Shape the plan first.

Turn this into a small implementation plan.

Look for: pages, components, data shape, smallest useful feature, and steps.

Slide 5

Demo 3: Add One Visible Feature

Use the starter project. The live demo feature is ingredient search, because people can see it working immediately.

Inspect this starter recipe website.
Add a simple ingredient search filter.
Keep the change small and explain what files you changed.
