# Facilitator Checklist

## Before Workshop Starts

- Confirm Codex download link is visible: `openai.com/codex`.
- Have the Recipe Finder starter project ready.
- Have the GitHub repo link ready: `https://github.com/ZhiweiChen-coder/codex-recipe-finder-starter.git`.
- Have a Google Drive zip backup ready.
- Remind attendees that no private or sensitive data is needed.
- Tell attendees they can use their own topic later, or continue with Recipe Finder.

## During Project Setup

- Show the starter project folder.
- Point out `README.md`, `PRODUCT_BRIEF.md`, `TASKS.md`, and `src/index.html`.
- Say that the starter has no install step.
- Ask attendees to open or download the starter before the demo begins.

## During Live Demo

Use the loop:

```text
Idea -> Plan -> Small Change -> Review -> Next Step
```

Demo sequence:

- Clarify the messy Recipe Finder idea.
- Ask for a simple plan.
- Inspect the starter project.
- Add ingredient search.
- Review the changed files and app result.
- Ask for a handoff summary.

## During Practice Time

Walk around and ask:

- What topic are you using?
- What did you ask Codex first?
- Did Codex give you a plan?
- What is the one small thing you are improving?
- What are you reviewing before accepting?
- What is your next step?

## If Someone Is Stuck

Suggest one of these:

```text
Use the Recipe Finder starter project and pick one task from TASKS.md.
```

```text
Make this simpler. What is the smallest useful next step?
```

```text
Explain what you need from me before continuing.
```

## Before Showcase

Ask for volunteers with different starting points:

- One Recipe Finder example
- One product or startup idea
- One docs, copy, or README example
- One technical repo example

## Keep The Room On Track

Say:

> We are not trying to build a perfect product. We are practicing the loop from rough idea to one useful, reviewable result.
