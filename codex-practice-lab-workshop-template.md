# Codex Practice Lab

## Session

Event: Codex Brisbane Community Night  
Facilitator: Peter  
Support: Jeremy, Peter Wei, NZ guest  
Time: 6:26 PM - 7:30 PM  
Length: 64 minutes  

## Workshop Goal

Learn the Codex Build Loop through one complete mini project.

The workshop should be:

- Easy for non-technical attendees to follow
- Concrete enough for developers to see a real project workflow
- Safe to run without private data
- Small enough to complete one visible change

## Mini Project

Use the Recipe Finder starter project.

Why it works:

- Non-technical users understand recipe websites
- It avoids sensitive business data
- UI changes are visible
- The same example covers idea, plan, copy, feature, README, review, and handoff
- Technical depth is controllable

Starter project:

```text
starter-projects/codex-recipe-finder-starter
```

Distribution:

- Primary: GitHub repo `https://github.com/ZhiweiChen-coder/codex-recipe-finder-starter.git`
- Backup: Google Drive zip `https://drive.google.com/file/d/1wpOzKtC5tVR1zqL6GckB1pvphOldMsKO/view?usp=drive_link`
- No setup required: open `src/index.html`

## Core Workflow

Everyone should practice this loop:

```text
Idea -> Plan -> Small Change -> Review -> Next Step
```

## Workshop Flow

### 1. Project Setup

Time: 6:26 PM - 6:31 PM

- Ask attendees to download Codex from `openai.com/codex`.
- Introduce the Recipe Finder starter project.
- Explain that attendees can use their own topic later, or continue with Recipe Finder.

### 2. Live Demo: Idea To Small Feature

Time: 6:31 PM - 6:46 PM

Demo steps:

1. Start with a messy product idea.
2. Ask Codex to clarify the idea.
3. Ask Codex for a small implementation plan.
4. Ask Codex to inspect the starter project.
5. Ask Codex to add one visible feature: ingredient search.
6. Review the changed files and app result.
7. Ask Codex for a short handoff summary.

### 3. Participant Practice

Time: 6:46 PM - 7:15 PM

Attendees can use:

- Their own product idea
- A website
- A README
- A workflow
- A GitHub repo
- The Recipe Finder starter project

Practice loop:

```text
Clarify -> Plan -> Improve one thing -> Review -> Summarise
```

If someone is stuck, suggest:

```text
Use the Recipe Finder starter project. Pick one small task from TASKS.md.
```

```text
Make this simpler. What is the smallest useful next step?
```

```text
Explain what you need from me before continuing.
```

### 4. Mini Showcase

Time: 7:15 PM - 7:25 PM

Invite 3-5 people to share.

Use this format:

1. What did you start with?
2. What did you ask Codex?
3. What did Codex produce?
4. What did you accept, change, or reject?
5. What is your next step?

Keep each share to 1-2 minutes.

### 5. Wrap The Learning

Time: 7:25 PM - 7:30 PM

Return to the loop:

```text
Idea -> Plan -> Small Change -> Review -> Next Step
```

Remind the room:

> The goal is not complexity. The goal is learning how to move from a rough idea to one useful, reviewable result.
